# Strategy Pattern と State Pattern

## 同じところ

Strategy (Policy) パターンと State パターンは、クラス図にすると同じ。
Strategy や State を切り出して、振る舞いをそちらの実装に任せる。

## 違うところ

Strategy が重視するのは、実装の互換性。
ある Strategy と別の Strategy の間には互換性があり、どちらを使ってもいい。
「戦略」という名前から分かる通り、Strategy の選択は状況によって変化しない。
Strategy によって呼び出し側のオブジェクトの性質を決める。
type と考えてもいいかも。基本的に一度決まったら変更しない。

State が重視するのは、元々のオブジェクトから状態の変化による動作の変化を切り出すこと。
状態に合わせて振る舞いを変化させることに重きを置いている。
Strategy のように振る舞いを実装に任せつつ、それが状態によって切り替えられることを示唆している。

設計意図が違うので別の名前がついている。
