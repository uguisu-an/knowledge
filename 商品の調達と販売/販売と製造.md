# 販売と製造

ビジネス

売る、
作る、
安く作る、
高く売る、
といった段階がある。

売るだけなら馬鹿でもできる。値段を下げればいい。

高く売れないなら、安く作るしかない。

仕事を取ってくるだけなら誰でもできる。なんでも 1 円で請ければいい。
そうではなくて、セールスの仕事は「限られた資源の中でより儲かるように、仕事を取ってくる」ことである。

顧客が「高く買っても元が取れる」と思えば、高く売れる。

「安く作る」ための基本は「断る」ことだ。
断りつつ、高く売るのが最良。

良いものを作るのもマーケティングの手段でしかない。

普通は早く売り始めた方が売れる。

売れるかどうか、だけではなくて、儲かるかどうか、を考えないと続けられない。

ビジネスの流れは大きく 3 つに分けられる。

- 購買前の「マーケティング」
- 購買を成立させる「セールス」
- 購買後の「カスタマーサポート」「カスタマーサクセス」

製品を作るのもマーケティングに含む。

プロダクトマネジメントは「売れるものを作る」に尽きる。
「作る」だけでも「売る」だけでも足りない。

「こういう機能があればユーザーに受けて、何ヶ月でこれくらいの収益が望める」というのも仮説。
こういう目標を立てて、それを検証する形のときにイテレーションは効果を発揮する。

サービスには、「使い続けてもらう」面と、「提供し続ける」面がある。
また、提供し続けるためには維持費を稼ぎ続ける必要がある。

受託開発は
企画 →（受注）→ 要件定義 → 設計開発 → サンプル製造 → 調整 → 納品
そもそも何を作るかも顧客に合わせる。開発がメインで量産しない。

オーダーメイドは、
企画 → 設計開発 → プロトタイプ製造 →（受注）→ 調整 → 納品
プロトタイプをベースに一人一人に合わせて調整する。調整がメインで量産しない。

受注生産では、
企画 → 設計開発 →（受注）→ 量産 → 納品
開発済みの既製品をまとめて受注してから量産する。

見込み生産では、
企画 → 設計開発 → 量産 →（受注）→ 販売
開発済みの既製品をあらかじめ量産して販売する。

セールスの場合は「問題を解決する」だけでなく「利益につながる」ことを説明する必要がある。

法人としての利益だけでなく、商談相手の個人としてのメリットを伝えないと「面倒だから今まで通りでいい」となってしまう。

開発や製造は、購入と同じように、販売や利用のための手段だ。
