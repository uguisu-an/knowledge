# Claude Code

Claudeをターミナル上で実行できる。
コードの読み書き、コマンド実行、コードベース全体の理解、マルチステップのタスク処理、MCPサーバーとの連携、などが特徴。

開発責任者はBris Cherny。

ローカルファイルにアクセスして読み書きできるのが最大の特徴。

Proプラン以上で使える。
使用量の制限があるので、ヘビーに使うなら従量課金が必要になるかも。

https://code.claude.com/docs/ja/overview

コマンド実行を制限する場合は設定ファイルに拒否ルールを設定できる。

大規模な処理の際に問題になるのが、コンテキストウィンドウやトークン使用量を圧迫すること。
https://code.claude.com/docs/ja/best-practices
https://code.claude.com/docs/ja/costs#reduce-token-usage

仕事でがっつり使う場合は月に100-200ドルかかると考えられる。
これより多かったら何か変なことをしていると疑った方がいい。

`CLAUDE.md`を設定すると、Codeの実行時に毎回読み込まれる。
ホームに置くと全てのプロジェクトに適用される。
Rulesはパスで指定して同じことができる。

CLAUDE.mdは使いたいプロジェクトのルートに置く。

普通に作業するならターミナル版でいい。
デスクトップ版だと使えないコマンドがあったりして制限がある。

「データ永続化の選択肢を裏で調査しておいて」のようなバックグラウンドタスクも指示できる。

`claude --continue` で前回の文脈を引き継いで起動できる。
特定の文脈を引き継ぎたかったら `claude --resume` でいい。

Claudeが動いているのを眺めるよりは、最初だけ質問などを待って、あとはたまに止まってないか確認する。
ただ、最初はどんなふうに動いているか眺めてもいい。

初めて使うMCPツールなんかは認証が必要になることもあるので最初は見ておいてもいい。

- `-dangerously-skip-permissions` は使わない。

何度も繰り返しそうな作業はとりあえず「コマンドにしといて」って指示しとけば大丈夫。

ルールやスキルを拡張することで精度を上げて使いまわせるから、webで使ってた時より可能性は感じる。
まぁ、webでも都度文脈を作り込んでやればかなり用途にあった動きはしてくれる。

`CLAUDE.md`を作るときはまずある程度Claudeと話してから`/init`で自動生成するといい。
