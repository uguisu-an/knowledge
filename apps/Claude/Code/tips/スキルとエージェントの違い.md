## Skillの強み

- 再現性・一貫性: 毎回同じ手順・品質で実行できる
- コスト効率: AgentはLLM呼び出しのループが増えるため、シンプルなタスクでは割高
- 速度: 推論ループを挟まないので速い
- 予測可能性: 何をするか明示的なので、デバッグ・監査がしやすい

例: 「PDFをdocxに変換する」「特定フォーマットでレポートを生成する」

## Agentの強み

- 曖昧なタスク: 目標だけ与えてアプローチを自分で考えさせる
- 動的な判断: 途中の結果に応じてステップを変える必要がある
- 複数Skillの組み合わせ: 何をいつ使うかをAgentが判断する
- 未知のエッジケース対応: 想定外の状況に柔軟に対処

例: 「この競合他社の状況を調べて、うちの戦略にどう影響するか分析して」

## 具体的な使い分け

- 手順が決まっている → Skill
- 品質を均一にしたい → Skill
- 何百回も繰り返す → Skill
- ゴールだけ決まっている → Agent
- 複数のSkillを組み合わせる → Agent（Skillを呼ぶ）
- 未知の状況に対応したい → Agent

AgentだけでSkillをゼロから毎回実装させると、品質がばらつき、コストが増え、遅くなる。
