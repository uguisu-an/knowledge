モデルとエフォートが上級になるほど上限に達するのが早くなる。
Sonnetの思考モードを使うのが基本。Opusは重要なタスクだけ。どうせOpusを使うならxhighでいい。maxはやりすぎ。

Claude自体は一連の指示と確認が終わったら `/clear` できるようにしたい。

- `/clear`: 文脈を破棄する
- `/compact`: 文脈を圧縮する

スクリーンショットなどで画像を扱うのも重い。
MVPの段階では基本的に人間が確認して、ある程度発展したら自動テストを整備するのがいい。

ずっとOpusで使っていると、「コミットして」みたいな単純作業でも余計なトークンを消費する。気軽に使うならSonnet中心に使った方がいい。
