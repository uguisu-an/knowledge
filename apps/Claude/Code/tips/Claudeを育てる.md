育成のポイント:

- あらかじめ想定された文脈を`CLAUDE.md`に追加する
- Claudeが何か間違ったことをしたのに気づいたら、それを`CLAUDE.md`に追加する
- `CLAUDE.md`もClaudeに編集させる
- `CLAUDE.md`をチームで共有する

文脈の例:

- 目的
- 背景
- 規約
- 制約

「同じ補足を何度も指示している」と思った時がわかりやすいサイン。
ルールやスキルに落とし込んで楽しよう。
最初から拡張しようと考えすぎなくていい。シンプルに普通の指示から始めて、必要に応じて拡張する。

Claudeが権限などのエラーで止まったら、自分で作業するのではなく、Claudeがそれを回避できるようにするための設定をClaudeに提案してもらう。

Claudeと話し合って、Claudeだけで完結できるようにするべきところ:

- 間違った作業
- 使用者に作業を依頼してきたところ
  - 権限がない
  - サンドボックスでエラーになる
  - など

指示一発でできないことも結構ある。
自分で違和感があるところは指摘して、Claudeと一緒に設定を細かく詰めた方がいい。スキルやサブエージェントなどもClaudeが作ってくれる。
