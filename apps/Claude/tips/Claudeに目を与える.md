「目」は「感覚器」のこと。

正常に完了できているかどうかをClaude側で確認できるようにする。

UIだったら、ChromeのプラグインなどでClaudeからUIを見たり動かしたりできること。
処理だったら、テストコードを用意して、パスするかどうか。

正常に完了しているか自分で確かめられるようになれば、Claude側で再試行して修正を繰り返せる。

Claude Code Desktopでは、組み込みのブラウザでスクリーンショットを撮って自動で確認してくれる。
ワイヤーフレームを渡しておくと精度をより高められる。

- テストスイート
- 組み込みブラウザによるUIの検証
- E2EテストによるUIの検証
- 「実装後に必ずテストを実行する」とルールに明記する
- Stopフックで全テストを実行する

テストが整備されていれば、テストが落ちるバグは「直せ」の一言で十分。
