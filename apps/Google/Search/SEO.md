# Search Engine Optimization

検索エンジン最適化

## SEOのポイント

1. 誰が、何を求めて検索したか、を一番に考える

- 結論から書く
- キーワードの背景を読んで、痒いところに手が届く内容にする

2. E-E-A-Tを意識する

- Experience (経験): 独自の一次情報を入れる
- Expertise (専門性): 特定のジャンルに特化する
- Authoritativeness (権威性): 他の信頼されるサイトからリンクされる
- Trust (信頼): 運営者情報を明記する、一次情報を引用する、など

3. 最低限の技術的な作法を守る

- タイトルにキーワードを入れる
- 文字の大きさや表示速度をスマホに合わせる
- 文書の構造を正しく使う

## 権威性を高める

- サイトの権威性を高める
  - 良い記事が、長い期間に渡って、多く投稿されている
  - より具体的でユーザーの知りたいことに合ったニッチな記事を増やす
  - 他の信頼されたサイトからリンクされている
  - 一つのジャンルに特化している
  - 権威性の高い投稿サイトにも投稿して、プロフィール欄から自分のサイトにリンクする
  - 検索数は少ないけどその人にとっては重要な問題をネタにする
- 投稿者の権威性を高める
  - 実務経験や保有資格なども権威性を補完する
  - ネット上で使っている名前を統一する
  - OSSなどに貢献して、貢献者リストに名前が載る

## その他

SEO が活きるのは、同じ検索ワードで出る他のサイトと比べて PV が多い時。
他のサイトも当然 SEO はしているはずなので、まずは PV を増やさないと順位は上がらない。
SNS とか広告を打つとか。

SEO は Web マーケティングの手段の一つでしかないので、まずは優先度の高いものからやっていけばいい。

- コンテンツ
- メタデータ
  - タイトル
  - 説明文
  - OGP (Open Graph Protocol)
- サイトマップ (`/sitemap.xml`)
- クローラー制御 (`/robots.txt`)

サイトマップとは、サイト上のページやファイルについての情報や各ファイルの関係を伝える情報。
このファイルを読み込むことで、検索エンジンがより効率的にクローリングできる。

サイトマップの定義。
ただし、Googleはpriorityとchangefreqを無視すると明言しているので、最終更新日が重要。
https://www.sitemaps.org/protocol.html

ページ単位で評価されて、サイト単位で信頼される。
ページの内容が良くても、どのサイトに属しているかで順位が大きく変わる。
これは信頼の積み上げがあるため。
質の悪い記事が増えると、サイトのレートも落ちてしまう。質の高い記事だけを置くべき。

サイト名で検索されるのはGoogle検索でも重要。

SEOを試す順序。

1. 記事を公開する
2. Google Search Consoleにサイトを登録する
3. 数週間後にコンソールでクエリを確認する
4. 表示回数が多いのにクリック数が少ないならタイトルや説明を改善する

SEO対策でトップを目指すなら、最低でも3-8週間はかかると思っていい。権威を高めるには時間がかかる。
自分でドメインを取った場合はさらに3-6ヶ月かかる。そもそも検索結果に表示されない。最初からSEOを試したいなら、Zennなどの投稿サービスを使う。

Zennの場合は技術記事が強いとか、投稿サービスによっても強いキーワードが違う。

「検索してもちゃんとした記事がなかった」を覚えておく。

検索キーワードは複数掛け合わせて、領域を絞ったほうが卓越しやすい。
"Next.js"よりは"Next.js AppRouter useSearchParams"のように絞って、キーワードを決める。

検索で勝ちたい場合、そのキーワードで検索して、上位の記事の質が低いかどうかを見る。
競合が弱かったら1位を狙える。

上位の記事に以下のような特徴があったら狙い目。

- 内容が古い
- 内容が薄い
- 記事が読みにくい
- 上位が投稿サービスの短い記事

1. 独自ドメインを使う
2. ドメイン名の長さは、あまり長くならないようにする
3. 将来的に多言語展開がある場合、.jp は使わない
4. サブドメインの利用は非推奨
5. URL（スラッグ）は、簡潔でわかりやすいアルファベット表記
6. URLは不変で、変わらないものにする
7. URLは静的なもので、パラメータにより区別されるものは極力避ける
8. 文字コードはUTF-8
9. 同一URLで、PC/スマホ両対応のレスポンシブサイト
10. SSL（https）対応
11. Googleビジネスプロフィールに登録する
12. 対象キーワードの検索ボリュームは、十分多いものを使う
13. 同じようなキーワード（検索意図が同じ、検索結果がほぼ同じ）はグループ化して、一つとして扱う
14. キーワードの種類（購入・行動につながるクエリと情報収集のクエリ）を考慮して、ページに割り当てる
15. 全国対象ではない場合は、地名を追加して確認
16. メインで狙うキーワードを一番最初に決める（社内相談必須）
17. 競合調査は、メインで狙うキーワードで検索した上位の類似サイトを調べる
18. 競合とは違う軸で勝てる要素をコンセプトにする
19. 主要なすべてのページにキーワードを割り当てる
20. サイト全体を親子孫の上下階層がある構成にする
21. トップページはサイト全体の目次として設計
22. グローバルナビを設置し、重要なページへの導線を確保する
23. フッターナビを設置し、主要なページへの導線を確保する
24. すべてのページがオリジナルの内容にする
25. 類似ページ、重複ページが多数になるような設計をしない
26. 狙うキーワードを極力、上部（ファーストビュー内が理想）に設置
27. コンテンツの下部、サイドバー、フッターなどから関連ページへのリンクを設置
28. 子供のページへのリンクは、コンテンツとして配置する
29. 長いページは最初に目次などを設置して内部リンクを設置
30. 200文字など極端に文字数の少ないページが出来ないようにする
31. タイトルとリンクのみのページは、極力作らない
32. できれば、カテゴリページ、一覧ページにも、リンク以外でオリジナリティを確保する
33. 1ページ＝1テーマにする
34. 狙うキーワードをコンテンツに含める
35. 狙うキーワードが複合キーワードある場合は、できる限り近接させる
36. 狙うキーワードの表記ゆれがある場合は、すべてのパターンを自然に入れ込む
37. 狙うキーワードを乱用しない。
38. キーワードの出現率を計算しない。
39. 関連キーワード、メインキーワードを意識して入れる
40. 狙うキーワードに対してユーザーのニーズを満たす内容を入れる
41. h1>h2>h3の構造を想定してライティングする
42. できれば。h2、h3などは、それぞれの親に対して3つ以上存在するようにする

44. h1とtitleタグには、狙うキーワードを前方に配置
45. h1とtitleタグの文字数は、長くしすぎない
46. h1とtitleタグは、ページ内容を適切に表すもの

48. h2〜h4を利用して構造化する
