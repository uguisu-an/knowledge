`git worktree` を使うと、一つのフォルダでブランチを切り替えて作業する代わりに、一つのブランチを一つのフォルダに対応させて作業できる。

## 新しいワークツリーを作る

`projects/my-project`というフォルダで作業していた場合、

```
git worktree add ../my-project-feature-1 feature-1
```

のように実行して、`feature-1`ブランチに対応したフォルダ（ワークツリー）を用意できる。

## `git clone`との違い

`git clone`と違って、`git worktree`は元の`projects/my-project`の`.git`を共有している。

- `.git`の分だけ容量を節約できる
- `git worktree list`でどのブランチをワークツリーにしているか見れる
- リモートリポジトリを介さなくてもブランチに対する変更を他のワークツリーでも確認してマージしたりできる
- すでにローカルにあるフォルダを展開するだけなので、`git clone`よりもセットアップが早い

## その他

長期的に変更を加えるようなブランチはワークツリーにしておくのが便利。
