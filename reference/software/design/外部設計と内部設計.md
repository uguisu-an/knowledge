# 外部設計と内部設計の違い

システムを外から見て、ユーザーとシステムの間のインタフェースを考えることを「外部設計」と呼ぶ。

システムを内から見て、システムの構成要素の間のインタフェースを考えることを「内部設計」と呼ぶ。

「システム」以外に、「UI」を中心に見れば、ユーザーと UI、UI と AP の境界が外部だし、 UI コンポーネントの区切りが内部だ。

内部とか外部とかいうのは、相対的なもの。
どこを基準にするのかいちいち明示しないと会話では使えない。

厳密に「UI コンポーネントの設計」とか「RDB テーブルの設計」とか言った方がいい。

システムの構成要素はサービスと呼ぶことがある。
システムに対して、なんらかの機能・役割を提供する。

システムの構成要素はそれもまたシステムとして捉えられる場合があるので、それにはさらに内部設計を行う。
あるシステムの内部設計は、その構成要素の外部設計になっている。

たとえば、以下のような構成のシステムがあるとして、

User --> UI : 1
UI --> AP : 2
AP --> DB : 3

普通は AP を中心に考える。
UI や DB を考えたり、インタフェース 2 (API) を考えたりするのは外部設計だ。
AP の中でオブジェクトや関数、ビジネスロジックなどを考えるのは内部設計だ。
