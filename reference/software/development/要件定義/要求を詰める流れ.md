要求の詳細を段階的に詰める流れ

前提
「誰が担当するか？」
「誰が利害関係にあるか？」
「いつまでにリリースしたいか？」
「どういうシステム構成にするか？」

課題
「どういう業務があるか？」
「どういう製品やサービスを提供したいか？」
「誰のためのサービスを提供するか？」
「困っていることはあるか？」
「この課題を解決するとどうなるか？」

// 「どの課題に対処するか？」

機能
「どの課題を解決しているか？」（効果）
「小さい機能に分けられないか？」
「この機能を使うのは誰か？」
「どういうことができるようになるか？」（利用者の使う言葉で記述する）
「開発にかかる時間や手間はどのくらいか？」
「技術的に実現できるか？」

// 「どの機能を開発するか？」（顧客と合意する）

仕様の概要
「どの機能を定義しているか？」
「どういう流れで操作するか？」
「どの画面で操作するか？」
「どうやって操作するか？」（能動的、リクエスト）
「いつ動作するか？」（受動的、イベント）
「どういうデータを扱うか？」（入力・出力項目）
「どういう動作を期待するか？」（表示、保存、送信、など）
「どういう条件で正しく動作するか？」（設定、データ、外部サービス、など）

// 「どの仕様を採用するか？」（顧客と合意する）

仕様の詳細
「どういう文言にするか？」
「どういう色や形にするか？」
「どこに配置するか？」

実装

// 「リリースできるか？」（顧客と合意する）
