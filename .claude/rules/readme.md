# フォルダ README.md の構成

`shared/`・`apps/`・`games/` 配下の各トピックフォルダ直下にある `README.md` は、そのフォルダが「どの概念を扱うか」を定義する。新しいノートをどのフォルダに置くか迷ったときは、候補フォルダの README を読んで境界を判断する。見本は `shared/development/README.md`。

## 構成

1. 概要文（見出しなし、1〜6行程度の短い散文）
   - その概念が何であるかを定義する
   - 可能なら、何ではないか（否定形の境界）も書く（例: `leisure/README.md` →「Business でないこと」）
   - 隣接・類似概念との関係や境界を書く（例: `design/README.md` → インタフェースと通信の関係）
2. `## Keywords`（任意）: 構成要素・関連語の箇条書き
3. `## References`（必須）: 親または関連する概念名の箇条書き。リンクは張らず概念名のみ。親子・兄弟関係が混在してよく、循環参照も許容する（`talk` ↔ `conversation` など）

フォルダ固有の注意事項がある場合は `## 編集方針` を追加してよい（例: `computer/development/README.md` →「『設計』に関することは『開発』に書かないようにする」）。

## 書くときの注意

- 概要文は短く保つ。詳細な知識は通常のノートに書き、README には境界判断に必要な情報だけを置く
- 隣接フォルダとの境界が曖昧になりがちな場合は、否定形の一文（「〜でないこと」）を概要文に加えて明示する
