# メモ取り込み

貼り付けられたメモを、このナレッジベースの適切な場所へ配置する。配置の分類規則・
ノートの書き方は @CLAUDE.md に従う。

## 手順

1. **ブランチを切る**。作業前に `memo/<YYYY-MM-DD>` のような作業ブランチを作成する
   （既に作業ブランチ上ならその旨を伝えて確認する）。
2. **メモを受け取る**。`$ARGUMENTS` のテキストを対象にする。空なら貼り付けを促す。
3. **配置先を判断するため**、関連フォルダの構造を確認する（`shared/` の分類、
   `apps/`・`games/` 等）。
4. 各メモに **識別子（連番）** を振る。
5. メモごとに **新規ファイル / 既存への追記** と **配置先パス** を判断する。
6. **除外対象を決める**（確認不要・自動除外）:
   - 公開リスクのある情報（実名を含む人物評、個人情報など）
   - コンテキストが不明で配置先を判断できないもの（URL のみなど）
   - 判断に迷うもの
     いずれも確認せずにそのまま除外とする。
7. **配置計画を一覧（識別子・要旨・配置先・新規/追記・理由）で提示**し、
   承認を待つ。**承認前に書き込まない。**
8. 承認後にまとめて配置する。
   - **新規ファイル**: 先頭に `# 日本語 English` 形式の見出しを付ける。
   - **追記**: 既存ファイル末尾に **空行1つ空けて** 原文を足し、**末尾にも空行**を残す。
   - メモ本文は **原文のまま**（内容を勝手に書き換えない）。
9. **除外したメモは原文のまままとめて最後にユーザーへ返す**（配置後・コミット前に返す）。
10. 配置後、ユーザーが内容を確認し手作業で調整する余地を残す。
    **指示があるまでコミットしない。**
11. コミット指示が出たら、リポジトリの慣習に合わせた簡潔な日本語メッセージで
    コミットする（末尾に Co-Authored-By 行）。
