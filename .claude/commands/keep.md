# メモ取り込み

貼り付けられたメモを、このナレッジベースの適切な場所へ配置する。配置の分類規則・
ノートの書き方は @CLAUDE.md に従う。

## 手順

1. **メモを受け取る**。`$ARGUMENTS` のテキストを対象にする。空なら貼り付けを促す。
2. **除外対象をメモの内容だけで判断する**（ファイルシステムへのアクセス不要）:
   - **行頭に 🚫 が付いたメモは無条件に除外する**（ユーザーの明示指定）。
     返すときは 🚫 を取り除いた原文にする。
   - 公開リスクのある情報（実名を含む人物評、個人情報など）
   - 個人的な抱負・日記・タスク・TODO など、公開知識として不適当なもの
   - コンテキストが不明で配置先を判断できないもの（URL のみなど）
   - 判断に迷うもの
   - 除外したメモは以降の手順から外す。確認不要・自動除外。
     除外したメモ（🚫 指定・自動判断とも区別せず）は**原文のまま**
     まとめてこの時点でユーザーへ返す（次の手順へ進む前に返す）。
     **引用ブロックは使わず平文で返し、複数あればメモごとに2行空ける。**
3. **ブランチを切る**。作業前に `memo/<YYYY-MM-DD>` のような作業ブランチを作成する
   （既に作業ブランチ上ならその旨を伝えて確認する）。
4. **配置先を判断するため**、`docs/folder-index.md` を読み込む。
   `shared/` への配置が基本。アプリ固有なら `apps/`、ゲーム固有なら `games/`。
5. メモごとに **新規ファイル / 既存への追記** と **配置先パス** を判断する。
   判断した配置先フォルダが実際に存在するか確認する（`ls` などで）。
   **存在しないフォルダが1つでもあれば** `/update-folder-index` を実行して索引を更新し、
   更新後の索引で配置先を再判断してからステップ7へ進む。
6. 配置計画に従ってメモを配置する。
   - **新規ファイル**: 先頭に見出しを付ける。
   - **追記**: 既存ファイル末尾に **空行1つ空けて** 原文を足し、**末尾にも空行**を残す。
   - 既存ファイル内に見出しがあれば適切な見出しの中に、適切な見出しがなければ `## その他` に入れる。
   - メモ本文は **原文のまま**（内容を勝手に書き換えない）。
7. 配置後、ユーザーが内容を確認し手作業で調整する余地を残す。
   **指示があるまでコミットしない。**
8. コミット指示が出たら、リポジトリの慣習に合わせた簡潔な日本語メッセージで
   コミットする。
