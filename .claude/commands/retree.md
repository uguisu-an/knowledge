# 概念ツリーの再構成

`docs/structure.md` の概念ツリーと `docs/folder-index.md` を、**各フォルダの README を情報源に**
ボトムアップで再構成し、再提案する。鳥瞰の道具。個別フォルダの中の区分は `/recat` の担当。
整理方針は @CLAUDE.md と @docs/organization.md に従う。

## この作業の前提

- **情報源は README だけ**（各 README の 概要・`## Map`・`## Keywords`）。9,000 個のノート本体は
  読まない。README が各フォルダの「定義」である以上、概念ツリーはその集約であるべき。
  これにより範囲を有界に保つ。
- **概念ツリーは README の派生ビュー**。両者が食い違うときは **README 側を正** とみなして
  ツリーを直す。ただし README 自体が薄い／ズレている場合は、直すべきフォルダとして指摘し、
  `/recat <folder>` の対象候補に挙げる（ツリーをその場で取り繕わない）。
- **範囲が広いので必ず2段階**。①棚卸し（診断）で一度止まり、②再構成提案へ進む。
  一気に書き換えない。
- **README の `## References`（ツリー上の親）は追加がデフォルト**。削除は手順5の
  Before → After に必ず明示し、承認を得てから行う（黙って消さない）。
- フラット優先。概念ツリーは「どう関連づけて考えているか」の参考であって、フォルダ階層と
  一対一ではない（structure.md の但し書き）。ツリーを精緻にしすぎて階層を深くしないこと。

## 手順

1. **範囲を受け取る**。`$ARGUMENTS` があればそのサブツリーに限定、無ければ `shared/` 全体。
2. **作業ブランチを切る**。`retree/<YYYY-MM-DD>` のような作業ブランチを作る
   （既に作業ブランチ上ならその旨を伝えて確認する）。
3. **棚卸し（フェーズ1）**。範囲内の各フォルダについて README を収集し、次を**報告して一度止まる**:
   - 各フォルダの 概要 と `## Map`／`## Keywords` の要旨（1行）
   - **README が無いフォルダ**（＝ツリーの欠損候補）
   - **Map／概念マップを持たない README**（＝情報が薄く、集約の素材にしづらい）
   - **フォーマットの不統一**（`## Map` / `## Keywords` / `## References` の混在など）
     ここで「欠損を先に埋めるか／現状の素材のまま再構成へ進むか」をユーザーに確認する。
4. **ボトムアップ集約（フェーズ2）**。各 README の主題を束ね、自然な上位カテゴリを作る。
   既存ツリーの `Being / Doing / Change …` といった枠は **建前として一旦外し**、README の
   中身から組み直す。既存の枠を残すかどうかも、中身に照らして判断する。
5. **提示して承認を待つ**。次を出す。**承認前に一切書き込まない。**
   - 概念ツリーの **Before → After**（新設・統廃合・移動した枝と、その理由）
   - `folder-index.md` との整合（定義が変わるフォルダ、索引から漏れているフォルダ）
   - **README 側を直すべきと判明したフォルダの一覧**（`/recat` 対象候補）
6. **（任意）承認後に更新する**。ユーザーが指示した場合のみ:
   - `docs/structure.md` の概念ツリーを書き換える。
   - `docs/folder-index.md` を実在フォルダと一致させる。
   - トップレベル構成（ルート `README.md` や `CLAUDE.md`）に波及するなら、その旨を指摘する
     （断りなく書き換えない）。
   - **フォルダの移動・新設・統廃合そのものは行わない**。それは `/recat` の承認フロー経由で行う。
     ここで扱うのは「ツリーと索引というドキュメント」の更新まで。
7. **原則レベルの発見を還元する**。再構成の過程で見えた一般則は `docs/organization.md` への
   追記候補としてユーザーに提案する（次回以降の再現性のため）。
8. **指示があるまでコミットしない**。コミット指示が出たら、リポジトリの慣習に合わせた
   簡潔な日本語メッセージでコミットする（末尾に Co-Authored-By 行）。
