# テストの範囲

## 概要

範囲が広いほど、「大きい」と言える。

範囲が広いと、壊れやすく（変更の影響を受けやすく）、メンテナンスコストが高い。
範囲が狭すぎると、数が増えやすく、メンテナンスコストが高い。

ほどほどに大きいテストでできるだけ広い範囲をカバーするべき。
条件が細かい計算などを小さいテストで、統合しないとテストできない部分だけ大きいテストでカバーする。

結合を剥がせないと単体テストできない。
単体テストすると疎結合を強いることになる。ただし、凝集されるとは限らない。
