keyにindexを使うのは最終手段。
まずはidを持たせることを考える。

以下の3つの条件をすべて満たしたらkeyにindexを使ってもいい。

1. 配列とその中の要素が静的(計算も変更もされない)
2. 配列の中の要素がidを持ってない
3. 配列がreorderやfilterされることが絶対ない

つまり、配列が別の位置に移動することがない（key=indexが変わらない）なら使ってもいい。

メモリー上にあるリストの編集機能を提供する場合、メモリー上であってもkeyとしてitemIdなどを自動生成して持たせた方がいい。keyがないと警告が出る。
