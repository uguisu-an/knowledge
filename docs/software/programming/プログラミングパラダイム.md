# プログラミングパラダイム

## 構造化プログラミング

実行位置の自由な移動 (GOTO 文) を禁止した。
代わりに、順次、反復、分岐だけの制御構造を用いる。

実行位置を好き勝手に移動できなくなったことで、問題・手続きを分割できるようになった。

## 関数プログラミング

代入を禁止した。
代わりに、変数を一度だけ定義する「束縛」を用いる。

変数の中身を変更できなくなったことで、並行処理で壊れるプログラムを書けなくなった。

## オブジェクト指向プログラミング

ポインタを禁止した。
代わりに、実装を隠す「インタフェース」を用いる。

インタフェースを対象に関連づけることで、既存の手続きを呼び出すのに実装を気にしなくて良くなった。
ただ、OOP 自体の実装がまだ十分ではなく、多くの抜け道ができてしまっている。
