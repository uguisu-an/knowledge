システム開発で考えること:

- 画面を使ったユーザーとの入出力
- 画面のレイアウト
- 画面に表示するテキストの内容
- データベースとの入出力
- 通常の処理
- バッチ処理
- ファイルとの入出力
- 帳票への出力
- 外部サービスとの入出力
- システムの物理・論理構成
- 仕様の検証・テスト方法
- 運用方法

早い段階でテスト計画を立て、実施していくことが全体的なコスト削減や価値提供の早期化につながる。
これを「シフトレフト」という。
https://zenn.dev/loglass/articles/b8b8b5688e91a2
