小さなレポジトリを一つの大きなレポジトリにまとめる方法をモノレポという。

特にマイクロサービスではレポジトリがたくさん作られるが、レポジトリが増えると更新やバージョン管理が煩雑になる。

privateな部分があるなら、全体をprivateにして、公開したい分だけ別リポジトリにもpushする手がある。
GitHub Actionsで特定ディレクトリを別のpublic repoに自動同期するワークフローを組む方法がよく使われる。

```
# .github/workflows/sync-public.yml
- name: Push to public repo
  run: |
    rsync -av --exclude=secret-core/ . ../public-mirror/
    cd ../public-mirror && git push
```
