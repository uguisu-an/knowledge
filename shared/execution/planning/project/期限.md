# 期限

プロジェクトにおいて、期限は絶対だ。
期限を守れないならプロジェクトとは言えない。

「1 時間でできるところまでやって」という指示もありうる。

期限を区切ることで、そこまでにできることだけを考える。

小さく問題を区切って、現実的にする。

期限が遠すぎると効果が薄い。
近い方は細かく、遠い方は荒く区切るといい。

大きな期限を細かく区切ったものをマイルストーンと呼ぶことがある。

期限に遅れるのが遅延。

複数の案件を同時に進めようとすると期限が必要以上に遠くなって、全ての案件の進行が遅れる。
これを防ぐには、案件をもっと細かく分割して、それぞれの作業の期限を一意に定めて、直列に繋ぎ直して一つずつ完了させる。

期限はできるだけ早めに設定するべき。できるかどうかわからないくらいギリギリがいい。その上で、超えてもいいように約束する。
期限を遠くすると完了もそれだけ遅れる。期限まで進めなくなる。
顧客に伝える期限と作業者に伝える期限は別々に設定するべき。これはマネージャーやディレクターを配置する意味の一つでもある。

いくら速く進めても、規模が大きすぎると期限は伸びる。
短期間で終わらせるためには、規模を小さく抑えないといけない。
