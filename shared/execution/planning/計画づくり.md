# 計画づくり

計画を立てること。

計画づくりは思考の一種。
計画も仮説でしかない。検証を進めて修正していく必要がある。

計画段階で特に重要なのは、何を観察するか。
観察の解像度が低いと学習効率も悪い。

計画を立てることで、実行時は一度に一つのことだけ考えればよくなる。
計画がないといちいち考えながら進めないといけないのでマルチタスクになりやすい。

計画づくりのベースになるのは課題、目的、イシュー。
イシューに対して、それを解決する作業をプロジェクトやタスクとして考えていく。

計画作りでも計画を何らかの形で成果として残すといい。
例えば、1日の計画を立てるときはチェックリストを成果にする。

https://dev.classmethod.jp/articles/load-to-pm-01-gather-all-todos/

> ものごとを計画するためには次の３つの条件を満たす必要があります。
>
> 1. **すべての**やることがわかっていること
> 2. **すべての**やることの優先度がわかっていること
> 3. **すべての**やることの締切がわかっていること

計画作りは論理的な作業。

計画づくりの段階でより良い計画を選ぶために、見積もりの能力が求められる。

計画を立てる前に、計画を立てることでどれくらいの仕事が減らせるか、素早く見積もらないといけない。計画を立てる意味がないなら計画は立てずにさっさと動き始める。

計画は、トータルの仕事量を減らすためにあるので、計画を立てる時間や手間でトータルの仕事量が増えてしまっては元も子もない。限られた時間の中で素早く計画しなければならない。

計画を立てるときは終わりから逆算する。

1週間の計画づくりと振り返りでも長期の計画について触れたい。
