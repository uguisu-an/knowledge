# PARA

実行可能性の高い順で分類するPARAメソッド
https://gigazine.net/news/20220509-para-method/

1. Project: 期限付きで目標にリンクされた一連のタスク
2. Area of responsibility: 長期にわたって維持されるべき基準を備えた活動領域（健康、財政管理、友人との関係、など）
3. Resource: 継続的に関心がある話題（習慣、趣味、仕事上のルーティン、など）
4. Archive: その他（特に完了したプロジェクトや維持する必要がなくなった領域やリソース）

Projectが最も実行可能性の高いところ。

特にプロジェクトと責任のある領域を分けて考えるのが大事。
責任のある領域にはタスクというよりも「維持すべき基準」がある。常に一定レベルの注意がある。その中にプロジェクトとして具体的なタスクがある。
領域内で基準から外れたものがあれば、それを修正するためにプロジェクトが生まれる。

責任のある領域だけを考えていてもうまくいかない。
具体的なプロジェクトに落とし込んで、達成していくことが重要。
