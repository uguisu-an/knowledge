# イベントとストリーム

リアクティブプログラミング。

ストリームは、（将来的に）出力される値のコレクションとして捉えられる。

ストリームは、イベントでもある。

> 「ストリームとは、時間順に並んだ進行中のイベントの列だ。」
>
> [【翻訳】あなたが求めていたリアクティブプログラミング入門 - ninjinkun's diary](https://ninjinkun.hatenablog.com/entry/introrxja)

Stream の 3 つのライフサイクルイベント

next ... 値を出力する
error ... エラーを出力する
complete ... ストリームを閉じる

ストリームの処理を説明するのに

----o----o----o------------o----x----|->

こういう表現を「マーブル図」という。

> RxJS = Observables + Operators + Schedulers.
>
> https://tech.recruit-mp.co.jp/front-end/rxjs-intro/

イベント駆動は OOP の時点ですでにあった。
というか、OOP が本来求めたのはイベント駆動、メッセージパッシングだった。

イベントリスナーは Command 系のメソッドで使うことが多くて、Query 系のメソッドでは使わない。
オブジェクトのプロパティでなくメソッドの引数に渡した方がいい。

ジェネレーターや yield はイベントストリームを扱うための構造。
同期でも非同期でも yield のタイミングでイベントが発行される。
