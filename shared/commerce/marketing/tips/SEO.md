# Search Engine Optimization

検索エンジン最適化

## SEOのポイント

1. 誰が、何を求めて検索したか、を一番に考える

- 結論から書く
- キーワードの背景を読んで、痒いところに手が届く内容にする

2. E-E-A-Tを意識する

- Experience (経験): 独自の一次情報を入れる
- Expertise (専門性): 特定のジャンルに特化する
- Authoritativeness (権威性): 他の信頼されるサイトからリンクされる
- Trust (信頼): 運営者情報を明記する、一次情報を引用する、など

3. 最低限の技術的な作法を守る

- タイトルにキーワードを入れる
- 文字の大きさや表示速度をスマホに合わせる
- 文書の構造を正しく使う

## 権威性を高める

- サイトの権威性を高める
  - 良い記事が、長い期間に渡って、多く投稿されている
  - より具体的でユーザーの知りたいことに合ったニッチな記事を増やす
  - 他の信頼されたサイトからリンクされている
  - 一つのジャンルに特化している
  - 権威性の高い投稿サイトにも投稿して、プロフィール欄から自分のサイトにリンクする
  - 検索数は少ないけどその人にとっては重要な問題をネタにする
- 投稿者の権威性を高める
  - 実務経験や保有資格なども権威性を補完する
  - ネット上で使っている名前を統一する
  - OSSなどに貢献して、貢献者リストに名前が載る

## その他

SEO が活きるのは、同じ検索ワードで出る他のサイトと比べて PV が多い時。
他のサイトも当然 SEO はしているはずなので、まずは PV を増やさないと順位は上がらない。
SNS とか広告を打つとか。

SEO は Web マーケティングの手段の一つでしかないので、まずは優先度の高いものからやっていけばいい。

- コンテンツ
- メタデータ
  - タイトル
  - 説明文
  - OGP (Open Graph Protocol)
- サイトマップ (`/sitemap.xml`)
- クローラー制御 (`/robots.txt`)

サイトマップとは、サイト上のページやファイルについての情報や各ファイルの関係を伝える情報。
このファイルを読み込むことで、検索エンジンがより効率的にクローリングできる。

サイトマップの定義。
ただし、Googleはpriorityとchangefreqを無視すると明言しているので、最終更新日が重要。
https://www.sitemaps.org/protocol.html

ページ単位で評価されて、サイト単位で信頼される。
ページの内容が良くても、どのサイトに属しているかで順位が大きく変わる。
これは信頼の積み上げがあるため。
質の悪い記事が増えると、サイトのレートも落ちてしまう。質の高い記事だけを置くべき。
