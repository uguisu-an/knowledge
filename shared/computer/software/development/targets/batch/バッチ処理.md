# Batch

一定期間データを集め、複数の処理をある程度まとめて一括処理を行う方式。

一定の単位でまとめて（バッチにして）、処理すること。

ユーザーが増えた結果、バッチ処理を実行するべき時間帯に処理が終わらないことを「突き抜け」ということがある。

https://www.imkk.jp/blog/introduction-batch-processing.html
https://engineering.mercari.com/blog/entry/2019-02-27-112650/

バッチの結果を収集する場合、できればバッチを呼び出すフレームワーク側に収集する機能があった方がいい。
バッチの中でデータベースやファイルに書き込む方法もあるが、バッチにバグがあると書き込まれないこともある。外にあった方がいい。
実行時のステータスコードは絶対に取れるので、バッチの外でステータスコードやエラー出力を記録する。
cronなどシンプルなものでも、cron側にステータスコードや出力の記録を書くといい。

- バッチ処理はいつまでに終わらないといけないか。
- バッチ処理に依存関係があるか。
