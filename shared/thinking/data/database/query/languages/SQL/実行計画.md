`EXPLAIN`をつけると実行計画を出力できる。

まず見るべきは、type, key, rowsのあたり。

- typeがallになっていないか（インデックスが使われているか）
- keyがnullになっていないか（インデックスが使われているか）
- 出力結果と比べて、必要以上にrowsが多くないか

そもそも出力する件数が多かったらrowsも多くなる。
出力する件数とrowsが同じなのが理想。

typeの種類:

- all: インデックスが使われていない
- const: 主キーで検索されている
- index: インデックスの全てが使われている
- range: インデックスの一部が使われている
