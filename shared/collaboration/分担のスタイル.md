1. フィーチャー型（機能横断）
2. レイヤー型（機能別）

| スタイル     | フィーチャー型   | レイヤー型         |
| ------------ | ---------------- | ------------------ |
| ユーザー価値 | 一つの作業で完結 | 統合して初めて完結 |
| 統合リスク   | 低い             | 高い               |
| 作業の範囲   | 広い             | 狭い               |

コーディングの場合、同じファイルに触ると作業が競合して、統合に手間がかかる。
同様に、同じものを変更する作業があると事務などでも同じように競合が発生する。
共通部分の扱いについては明確にしておくべき。例えば、インフラはインフラチームに委託する形で常にインフラチームだけが触る、というような運用も多い。
