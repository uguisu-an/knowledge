# 指揮

Conduct, Command

「指揮」とはタイミングを合わせること。

タイミングを合わせる必要もなければ、指揮も必要ない。

タイミングを見極める必要がないなら、指揮官は必要ない。

演奏の得手不得手は指揮者の関知するところではない。タイミングが合っていれば。

指揮官を置くのは、それだけ「タイミングを見極める」のが重要であることを示している。

リーダーと指揮官は違う。

マネージャーと指揮官は違う。

指揮と指示は使い分ける。
着手するタイミングをコントロールするのが指揮で、作業の内容にまで踏み込むのが指示だ。
作業の素人でもない限り、指示する必要はない。指示が必要なのは、育成の時だけ。

指揮者は指揮よりも前に信頼関係を構築しないといけない。ただ、その上で指揮能力もないと害悪になる。

指揮する側の仕事をしていると、「他人をコントロールできる」という妄想に囚われる人がそこそこいる。

組織やチームではタイミングを合わせるのが大事。
組織では同じタイミングが基本、チームではもっと複雑に組み合わせる。

指揮を始める前に、誰の指示に優先して従うべきか、指揮系統を部下全員に周知しないといけない。
誰に従えばいいのかわからないと判断が遅れる。

上司は、その上司より下の上司が全員同じ指示を出すように指示を一貫させる。下の上司をクビにしたり、教育したり。いちいちズレていると判断が遅れる。
指示が一貫してないということは、下の上司が上の上司の指示を聞いていないということ。上司が指示を聞いてないのに部下が指示を聞くわけがない。

指揮はより早い判断、ひいては行動のためにある。集団の動きを早くするために指揮を取る。指揮系統があっても動きが遅いなら統治に失敗している。
