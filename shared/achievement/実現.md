# 実現

implement

「できない」かどうかは短期計画に落とし込んでみてから考えればいい。

答えはいくつもある。考える時点では結論を出さなくてもいい。あとで絞ればいい。

頭の中にはいくつも答えがあっても、実現できるのはとりあえず一つずつとか優先順位が必要。作るのにコストがかかる。

思いつきを形にするのに一番簡単なのは、専門家に頼ること。

workが先、achievementが後。
「変化させる」のがまずあって、その方向性を制御する。

achievementは問題解決に近い。

成功の一種。

実現や実装という活動は、エラーを取り除いていく活動。
まず理想と現実の差を見つけて、埋めていく。

理想を実現するには、現実の状態を把握して、改変して、さらに最新の状態を把握し続ける必要もある。

問題解決とか課題管理とか全部achievementにまとめてもいいかも。

大きな理想を実現にするには、人や機械の手を借りて、過去の偉人の肩に乗って、とさまざまな手段を使う。
