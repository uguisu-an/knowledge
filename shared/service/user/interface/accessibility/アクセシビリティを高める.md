## 方針

- 情報を構造化する
- インタラクションを標準化する
- 認知負荷を軽減する

## 具体的な対応例

- 適切なalt属性をつける
- 装飾目的の画像には空のalt属性を明示する
- inputとlabelを関連づける
- 動画に字幕をつける
- キーボード操作でフォーカスが当たる
- ページ遷移したらフォーカスを適切な場所に移す
- 色だけで情報を伝えない
  - アイコンやメッセージをつける
- 色弱でも見やすいコントラスト比を確保する (4.5:1が目安)
- 見出しを正しく使って階層化する
- 意味のある要素（セマンティックタグ）を活用する
- `aria-describedby`でinputとエラーメッセージを関連づける
- 読み上げ、スクリーンリーダーに対応する
- `aria-live`や`aria-busy`を使ってスクリーンリーダーにも読み込み状態を伝える
- 複雑な文章を避けて簡潔な言葉を使う
- ユーザーの記憶力に頼りすぎないUI設計にする

## 対応の優先順位

1. セマンティックHTML
2. フォームのラベル
3. キーボード操作

## アクセシビリティの確認方法

- キーボードだけで操作する
- スクリーンリーダーを使って読み上げる
