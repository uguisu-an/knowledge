# Accessibility

a11yと略されることもある。

どんな人でもその情報にアクセスできることを目指す。

アクセシビリティで一部の人の使い勝手を良くする前に、すべての人にとっての使い勝手を良くするべき。アクセシビリティはあくまでも補助。

https://qiita.com/kobayashimakoto/items/2aa2bbcf6d47b697e3e6

https://developer.mozilla.org/ja/docs/Learn_web_development/Core/Accessibility

キーボード操作のアクセシビリティ対応はフォーカス管理。

「ランドマーク」はページの領域を示す。navだったり、mainだったり。大きな範囲で使う。
https://developer.mozilla.org/ja/docs/Learn_web_development/Core/Accessibility/WAI-ARIA_basics#%E9%81%93%E3%81%97%E3%82%8B%E3%81%B9%E3%83%A9%E3%83%B3%E3%83%89%E3%83%9E%E3%83%BC%E3%82%AF

UIコンポーネントのライブラリを使うとa11y周りも標準で対応されている部分が多くて楽。
