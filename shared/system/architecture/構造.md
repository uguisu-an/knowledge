# Structure

システム全体に対して、部分同士の関係を構造という。

構造は主に静的な部分。
システムとしてはその中でやり取りされる情報や捜査などの動的な部分も重要。

- 依存関係・前後関係
- 階層構造・親子関係
- 抽象と具象
- 概要と詳細

インタフェースによるコンポーネントの繋がりの全体。

外観もまた構造である。

多くのものには「こうすると、こうなる」という反復構造がある。

構造を理解するには言葉を理解するのが簡単。

構造を考えるということは、情報を知識にするということ。

全体として、構成要素とその関係（構成）がある。

- どんな構成要素があるか
- それらの要素はどんな関係にあるか
- どうやって構造を記述するか
