# アジャイルプラクティス

「アジャイル」は名詞ではない。

「アジャイル」という手法があるわけではない。

「アジャイル」は一般的な形容詞だ。

単に「迅速」なことを表している。

アジャイルプラクティスは、アジャイルな「状態」に至るための「練習」だ。機動性そのものではない。

スクラムや XP は「アジャイル」ではなく「アジャイルプラクティス」であり、アジャイルなチームに至るための修行方法である。

アジャイルは結果として現れる「状態」であって「過程」ではない。

スクラムなどのアジャイルプラクティスは、「アジャイルになるための」プロセスであって、「アジャイルな」プロセスではない。

アジャイルプラクティスの技術面は、イテレーションかどうかに依存しない。
継続的デリバリー、自動テスト、スモールリリース、ホールチーム、ドメインモデルなど、開発の密度を高めるための活動は全てに通ずる。
