そもそも名前の重複を自動で回避しなくても、重複エラーで十分だと思う。

ファイル名の重複を回避する必要があるのは、新しいファイルを一旦デフォルトの名前で保存するときだけ。

ファイル名の重複を回避して、新しいファイル (1) みたいにつける場合、クライアント側で処理するか、サーバー側で処理するか。
サーバー側はできるだけシンプルにしたいと思ってる。重複してたら弾くだけ。同じタイミングで処理した場合に同名で保存しようとして整合性でエラーになることも考えられる。結局、エラーはありうる。
クライアント側だと、Web API を何度も叩くのは効率が悪い。

中間択として、名前が重複したらサーバー側で「候補を生成する」API を叩くのもありかも。
エラーが返ってきたら再試行する形。単に重複エラーで落とす形に変えても API の変更が必要ない。責務が綺麗に分かれていてテストしやすい。ただ、途中で画面を閉じられると保存されない可能性がある。
