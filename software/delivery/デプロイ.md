開発サーバーや本番サーバーに移行するソフトウェアのバージョンは、GitHub にあるいずれかのバージョンのうち、バージョンタグが割り当てられているものに限るべき。
Git に入っていないもの、一部を Git の外で改変したもの、バージョンタグが割り当てられてないものなどは禁止する。
特に本番サーバーはメインブランチのものだけ。開発サーバーは feature ブランチなども可。

デプロイはできるだけ自動化した方がいい。

デプロイツールは fabric や capistrano が有名だったけど、最近はどっちも落ち込んでいる。
単純に ssh か、ansible でデプロイまでするとこが増えてるのかも。
